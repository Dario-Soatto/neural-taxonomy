\begin{itemize}
  \item Split/remove thresholds compared \(L_z\) against \(\log p(x)\)-scale baseline (\(\approx -35\)).
  \item Result: thresholds too loose in practice for split/remove; no split/remove triggered.
\end{itemize}

\textbf{After revision (run \texttt{em\_14338338}):}
\begin{itemize}
  \item \(L_z\) and decision baseline are aligned on \(\log p(x\mid z)\)-scale
  \item Decision baseline moved to \(-1.7211\) (from \(-35.1943\) on raw \(\log p(x)\))
  \item Split/remove thresholds became reachable and were crossed.
\end{itemize}
\end{frame}

\begin{frame}{Next Steps (Beyond Duplicate-Label Bug)}
\small
\begin{itemize}
  \item Add \textbf{accept/reject gating} for structural moves using held-out AIC/BIC/ELBO deltas (not just heuristic triggers).
  \item Introduce \textbf{semantic relabeling for split children} (avoid inherited duplicate names and improve interpretability).
  \item Add a \textbf{posterior sanity check}: sample items \(x_i\) from a target label \(z_j\) and verify
  \(p(z_j \mid x_i) > p(z_k \mid x_i)\) for most \(k \neq j\).
  \item Evaluate stability across hierarchy levels (e.g., 8/10/12 clusters) and multiple random seeds.
  \item Add per-action ablations: add-only vs split-only vs split+add to isolate which move drives score gains.
  \item Add post-EM external quality checks (coherence, downstream relabeling accuracy, human spot checks).
\end{itemize}
\end{frame}
